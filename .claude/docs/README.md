# Internal Documentation

This directory contains internal development and testing documentation for the Unicon project.

## Contents

### Implementation & Architecture
- **IMPLEMENTATION_SUMMARY.md** - Technical implementation details and architecture decisions
- **DOCS_SUMMARY.md** - Overview of project documentation structure

### Testing & Quality
- **MCP_TESTING_CHECKLIST.md** - Comprehensive testing checklist for MCP server functionality
- **Kaizen.md** - Continuous improvement notes and refactoring tasks

### Publishing & Deployment
- **PUBLISH_MCP_SERVER.md** - Steps and procedures for publishing the MCP server
- **MCP_SERVER_PUBLISHED.md** - Publication status and post-publish notes

## Purpose

These documents are kept separate from the main README to:
- Keep the root directory clean for public contributors
- Maintain internal context for maintainers and AI assistants
- Document testing procedures and deployment processes
- Track technical decisions and improvements

## For Contributors

If you're contributing to Unicon, you typically don't need these docs. Refer to:
- **../README.md** - Main project documentation
- **CLI.md** - CLI usage documentation
- **docs/** - Public-facing documentation

## For Maintainers

These documents provide essential context for:
- Testing new features
- Publishing releases
- Understanding architectural decisions
- Planning improvements
